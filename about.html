About

What is SQUISH 'EM!?

A lightweight, dependency-free HTML5 arcade game built entirely with vanilla JavaScript, HTML5 Canvas, and the Web Audio API.

The Game

SQUISH 'EM! is a mobile-first blob shooter inspired by classic vertical arcade games. Kawaii-style enemies fall from the top of the screen, and your job is to blast them with your sliding cannon before they reach the bottom.

The game features progressive difficulty scaling, a combo system, boss battles every 5 levels, four distinct power-ups, and a locally-saved high score. Despite all of this, it runs as a single HTML file with zero external dependencies.

Technology Stack

🖼️
HTML5 Canvas API
All rendering — blobs, particles, UI, animations — is drawn each frame using the 2D Canvas API. No DOM elements in the game view.
🔊
Web Audio API
Every sound effect is synthesised at runtime using OscillatorNode and GainNode. No audio files are loaded — zero network requests for sound.
⚡
requestAnimationFrame
The game loop uses rAF for a stable 60fps target. Delta-time capped at 50ms prevents spiral-of-death when the tab is hidden.
💾
localStorage
High scores persist across sessions using the browser's localStorage API — no server or account needed.
📱
Pointer Events
Touch and mouse events are unified with canvas-coordinate scaling so the game works at any screen size and pixel density.
🎨
Pure CSS Layout
The surrounding website is built with pure CSS Grid and Flexbox — no frameworks, no preprocessors. A single style block per page.

How It Works — Architecture

Game Loop

The engine runs a fixed requestAnimationFrame loop. Each tick calculates delta-time (dt in seconds, capped at 50ms), then calls update(dt) followed by a full canvas redraw. All physics — positions, timers, animations — are scaled by dt to stay frame-rate independent.

Object Pools Performance

Bullets, particles, power-ups, score popups, and boss projectiles are held in plain JavaScript arrays. Objects are spliced out when they expire rather than using a formal pool, keeping allocation low. The maximum active blob count is capped per level (5, 8, then 12) to prevent runaway spawning.

Blob Rendering

Each blob is drawn procedurally every frame using Canvas arc and quadraticCurve calls:

  • Drop shadow ellipse (offset 2px, semi-transparent)
  • Main body ellipse (colour per type, white flash on hit)
  • Highlight oval (top-left, 35% white alpha)
  • Eyes: dot arcs and pupils; expression variants for sleepy, angry, and smug
  • Mouth: quadratic bezier curve (happy, angry, smug variants)
  • Wobble: sinusoidal Y-scale ±5% driven by a per-blob phase counter
  • Death: dScale shrinks to 0 while inverse Y-scale expands — the classic squish effect

Procedural Audio

All audio is synthesised on the fly using the Web Audio API:

  • Shoot — short 880 Hz sine wave, 70ms, low volume
  • Squish — 220 Hz sawtooth + 180 Hz square with -200 cent detune
  • Power-up — ascending four-note chime (440, 550, 660, 880 Hz)
  • Life lost — 150 Hz sawtooth with -300 cent detune + screen shake
  • Level up — C-E-G-C fanfare using triangle oscillators
  • Boss appear — 60 Hz sawtooth, 1 second, high volume
  • Game over — descending five-note tune (C, B, A, G, F)

Canvas Scaling

The logical resolution is fixed at 400x700px. On mount and window resize, a scale factor is computed as min(window.innerWidth/400, window.innerHeight/700). The canvas CSS dimensions are set to logical * scale, and all input coordinates are divided by scale to convert back to logical space. The game looks sharp on any screen without touching game logic.

Collision Detection

All collisions use circle-to-circle distance checks: sqrt(dx*dx + dy*dy) < r1 + r2. Since blobs are approximately circular and bullets are small, this is visually accurate. No spatial partitioning is needed at the blob counts this game targets (under 12 simultaneous enemies).

FAQ

Does this game require an internet connection?
The game runs entirely client-side. Once the page is loaded, it works offline — no API calls, no CDN assets, no analytics.
Where is my high score saved?
In your browser's localStorage under the key squish_hi. It persists between sessions on the same browser and device. Clearing browser data will reset it.
Why can't I hear any sound?
Most browsers require a user interaction before allowing audio. Tap or click the canvas once to start the game and audio will activate. Check that your device is not muted.
Is the source code available?
Yes. Full source is on GitHub — all game logic, rendering, and audio in a single HTML file. View source in your browser or check the repository below.
Does it work on mobile?
Yes. The game is mobile-first. Designed for portrait touch screens and tested on iOS Safari and Android Chrome. Drag controls and on-screen buttons are optimised for thumbs.
Is there an end to the game?
There is no final level — the game scales infinitely. Blob speed is capped at 300% base speed, but more blobs spawn simultaneously. Survive as long as you can and maximise your score.
